# Xhttpx - 批量域名存活探测工具

> 一款基于 Python 开发的批量探测域名存活状态的工具，支持 txt、csv、xlsx 等多种格式的域名列表文件，采用多线程并发探测，高效快速验证域名是否可访问，并区分存活域名、不可访问域名以及需要手动高级连接的域名。

## 🌟 功能特点
- 支持多格式文件读取：txt/csv/xlsx/xls 格式的域名列表
- 多线程并发探测：可自定义线程数，默认 150 线程，探测效率高
- 智能协议重试：HTTPS 请求失败自动尝试 HTTP 协议
- 详细的探测结果：返回响应状态码、页面长度、页面标题、存活状态
- 结果分类保存：存活域名、需要手动高级连接的域名分别保存至不同文件
- 自动去重：对读取的域名列表自动去重，避免重复探测
- 友好的终端输出：彩色字体区分不同探测状态，直观易读

## 环境要求

- Python 3.x

- 所需依赖包：

  ```text
  requests
  xlrd
  urllib3
  ```

## 使用方法

1. 准备域名列表文件（支持 txt/csv/xlsx 格式，域名需放在第一列）

2. 运行工具

   ```python
   python xhttpx.py
   ```

3. 根据提示输入待探测的文件路径（如：`domains.txt`、`urls.csv`、`hosts.xlsx`）

4. 等待探测完成，结果会自动保存至当前目录下的文件

### 文件格式示例

- TXT 格式：每行一个域名

  ```
  example.com
  github.com
  test.com
  ```

- CSV/XLSX 格式：第一列为域名，支持表头（自动过滤空值和 None）

  |                域名                |
  | :--------------------------------: |
  | [example.com](https://example.com) |
  |  [github.com](https://github.com)  |

## 配置说明

可在代码中修改以下核心配置：

```python
# 并发线程数（根据服务器/网络环境调整）
thread_num = 150
# 请求超时时间（秒）
timeout = 5
# 结果保存文件名
result_file = "存活域名.txt"
# 请求头（可自定义UA）
headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:148.0) Gecko/20100101 Firefox/148.0'
}
```

## 📊 输出结果说明

- `存活域名.txt`：保存可正常访问的域名（响应状态码 200/301/302）

- `高级访问.txt`：保存请求失败（隐私错误）需要手动高级连接的域名

## 注意事项

1. 线程数建议根据自身网络环境调整，过高可能导致请求失败率上升

2. 部分域名可能因反爬策略、网络限制等因素探测结果不准确

3. xlrd 库对.xlsx 文件的支持可能存在版本兼容问题，若报错可安装指定版本：

   ```python
   pip install xlrd==1.2.0
   ```

4. 工具仅用于合法合规的域名探测，禁止用于未经授权的测试

## 🎨 终端输出状态说明
- 🟢 绿色：域名存活，可正常访问
- 🔴 红色：域名死亡，响应状态码非 200/301/302
- 🟡 黄色：请求失败，需要手动高级连接
