# Xhttpx - 批量域名存活探测工具

> 一款基于 Python 开发的多线程批量探测域名存活状态的工具，支持 txt、csv、xlsx 等多种格式的域名列表文件，自动尝试 HTTPS/HTTP 协议，快速识别存活域名并输出结果。

## 功能特点

- 🚀 **多线程并发**：自定义线程数，大幅提升探测效率
- 📂 **多格式支持**：支持 txt、csv、xlsx/xls 格式的域名列表文件读取
- 🔌 **双协议检测**：优先 HTTPS 探测，失败自动重试 HTTP
- 🧹 **自动去重**：对读取的域名列表自动去重，避免重复探测
- 📝 **结果保存**：自动将存活域名保存到本地文件，便于后续使用
- 🎨 **彩色输出**：终端彩色标识存活 / 死亡状态，直观易读
- ⏱️ **耗时统计**：显示总探测耗时，便于评估效率

## 环境要求

- Python 3.x

- 所需依赖包：

  ```text
  requests
  xlrd
  urllib3
  ```

## 使用方法

1. 准备域名列表文件（支持 txt/csv/xlsx 格式，域名需放在第一列）

2. 运行工具

   ```python
   python xhttpx.py
   ```

3. 根据提示输入待探测的文件路径（如：`domains.txt`、`urls.csv`、`hosts.xlsx`）

4. 等待探测完成，结果会自动保存到`存活域名.txt`文件中

### 文件格式示例

- TXT 格式：每行一个域名

  ```
  example.com
  github.com
  test.com
  ```

- CSV/XLSX 格式：第一列为域名，支持表头（自动过滤空值和 None）

  |                域名                |
  | :--------------------------------: |
  | [example.com](https://example.com) |
  |  [github.com](https://github.com)  |

## 配置说明

可在代码中修改以下核心配置：

```python
# 并发线程数（根据服务器/网络环境调整）
thread_num = 150
# 请求超时时间（秒）
timeout = 5
# 结果保存文件名
result_file = "存活域名.txt"
# 请求头（可自定义UA）
headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:148.0) Gecko/20100101 Firefox/148.0'
}
```

## 注意事项

1. 线程数建议根据自身网络环境调整，过高可能导致请求失败率上升

2. 部分域名可能因反爬策略、网络限制等因素探测结果不准确

3. xlrd 库对.xlsx 文件的支持可能存在版本兼容问题，若报错可安装指定版本：

   ```python
   pip install xlrd==1.2.0
   ```

4. 工具仅用于合法合规的域名探测，禁止用于未经授权的测试
